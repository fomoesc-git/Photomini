**项目时间**: 2026年4月
**技术栈**: Tauri 2.x + React + TypeScript + Rust
**目标平台**: macOS (Apple Silicon)
**项目路径**: ~/projects/Photomini图片压缩工具/

## 项目概述

一个面向日常图片处理的桌面工具，支持：
- 图片压缩（JPG/PNG，保持尺寸）
- 格式转换（JPG ↔ WebP）
- 尺寸调整（最长边缩放）

## 踩坑记录

### 1. Rust 命令函数不能是 `pub`

```rust
// ❌ 错误
#[tauri::command]
pub fn compress_image(...) -> ProcessResult { ... }

// ✅ 正确
#[tauri::command]
fn compress_image(...) -> ProcessResult { ... }
```

**原因**：`pub` 会导致宏展开后产生重复符号 `__cmd__compress_image`

---

### 2. `file_stem()` 丢失父目录

```rust
// ❌ 错误 — 输出到当前目录而非原目录
let stem = p.file_stem().unwrap();
let output = format!("{}.webp", stem);

// ✅ 正确
let parent = p.parent()
    .map(|pp| pp.to_string_lossy().to_string())
    .unwrap_or_default();
let stem = p.file_stem().and_then(|s| s.to_str()).unwrap_or("output");
let output = if parent.is_empty() {
    format!("{}.webp", stem)
} else {
    format!("{}/{}.webp", parent, stem)
};
```

---

### 3. macOS 窗口关闭死锁

```rust
// ❌ 错误 — 在 window_event 回调中同步调用 close() 导致主线程死锁
w.on_window_event(move |event| {
    if let tauri::WindowEvent::CloseRequested { api, .. } = event {
        api.prevent_close();
        ww.close().unwrap(); // DEADLOCK!
    }
});
```

**原因**：macOS 的 WebView 窗口 close() 在窗口事件回调的同一线程执行，而该回调持有窗口锁，造成死锁。

---

### 4. `app.listen()` 闭包捕获 `&mut app` 问题

```rust
// ❌ 错误 — app 含有非 Send 类型，闭包不是 Send，编译报错
app.listen("tauri://activate", move |_event| {
    let handle = app.handle().clone(); // 捕获了 &mut app
});

// ✅ 正确 — 用 handle.clone().listen()
.setup(|app| {
    let handle = app.handle().clone();
    handle.clone().listen("tauri://activate", move |_event| {
        if handle.get_webview_window("main").is_none() {
            // rebuild window
        }
    });
    Ok(())
})
```

**原因**：`app.listen()` 要求闭包 `Fn(Event) + Send + 'static`，而 `&mut app` 持有 `Rc<EventLoop>` 等非 Send 类型。

### 5. `devtools` 配置字段在 Tauri 2.x 不存在

```json
// ❌ 错误 — 未知字段
{
  "build": {
    "devtools": false

// Tauri 2.x 中移除此字段，devtools 由运行时控制
```

## 关键教训

### 1. 立即验证而非假设正确
这次开发中期存在"汇报完成但未实际执行构建"的问题。以后每次修改 Rust 代码后应立即运行：

```bash
npm run tauri build --bundles dmg
```

而不是假设代码逻辑对了就没问题。

---

### 2. macOS 窗口行为复杂度高于预期
Tauri 2.x 的窗口重建涉及线程安全约束，比想象中复杂。如果不是刚需，不要过度设计窗口行为。

**最终结论**：在 macOS 上，点关闭按钮直接退出是最简单可靠的行为。

---

### 3. bundle identifier 规范
macOS bundle ID 不能以 `.app` 结尾：

```json
// ❌ 错误
"identifier": "com.photomini.app"

// ✅ 正确
"identifier": "com.photomini"
```

---

### 4. Rust 路径处理
始终检查 `parent()` 是否为空：

```rust
let parent = p.parent()
    .map(|pp| pp.to_string_lossy().to_string())
    .unwrap_or_default();
```

避免文件写入到当前工作目录而非原目录。

## 后续改进建议

如果未来要重新做窗口隐藏 + Dock 点击重建功能：

1. 考虑不拦截 `CloseRequested`，改为监听其他生命周期事件
2. 或者在 macOS 上使用 `NSWindow.hide()` 而非关闭窗口
3. 窗口重建前先 `app.get_webview_window()` 判断是否已存在
4. 使用 `handle.clone().listen()` 而非 `app.listen()`
